App's Lifecycle with CICD

Intro

CICD is a set of rules, requirements, good practicies and tools for working on software projects. Thanks to CICD, dev team is able assure frequent delivery of certain and tested changes to the code.

Modern CICD solutions provide:
- continuous integration,
- continuous testing,
- continuous delivery,
- continuous deployment,
- continuous monitoring
through entire software development lifecycle.

CICD is a compromise betweem developers who want to make changes to the code frequently and server admins whose aim is to keep server and application stable. Therefore, CICD ensures easiness of code changes with app stability simultaneously. Automation of delivering changes to the code and automated testing make process of deploying changes to the server fast and reliable.

Continuous Integration:
  • Most of modern apps requires different techonologies when creating a code. This is why dev teams need a mechinizm for integrating an compiling the application.
  • A good practice is when developers commit their code to version repository frequently - at least one time per day.
  • That smmall chunks of codes are much easier to analyze and debug as opposed to a big block of code.
Continuous Testing:
  • Using special frameworks to perform auotmated tests.
  • Testing app or its fragments if they work properly due to assumptions.
  • The most important test is regression testing of whole application in the context of provided changes.
  • Regression testing is basically re-running functional and non-functional tests on the application in order to make sure that recently developed and tested software still performs after a change. If test fails then we have a regression.
  • The best practice is to perform regression tests in local environment before commiting application into remote repository.
Continuous Delivery:
  • It comes when app built, validated and tested out to release the code to a remote repository.
  • It automates process of providing app and inserting code changes into repository.
  • It just makes sure that the app is alwyas in a production-ready state - always ready to be released.
  • Teams, using clodud solutions, deliver their app in containerizated form with Docker or Kubernetes.
Continuous Deployment:
  • Whereas continuous delivery automates the release code in a production-ready state to the repository, continuous deployment automates releasing an app to production.
